## 1. What This Kit Is

This is a universal production-grade agent kit for Google Antigravity.
It works for any project type — web apps, mobile apps, APIs, ecommerce platforms,
inventory systems, SaaS dashboards, and more.

Drop this kit into any project. Run `/start`. The agent handles everything else.

## 4. UX Research Files — Read When Present

Before any UI design or build session, check the project root for research files:

```
Look for any of these patterns in the project root:
- ANALYSIS-*.md
- *-NEW-DIRECTION.md
- *-UX-RESEARCH.md
- *-DESIGN-DIRECTION.md
```

If any are found, read them before starting any UI work.
These files contain deep UX analysis of reference sites and proposed visual
directions. They are the single most important source of UI intent for the project
and override any generic assumptions about how the UI should look or feel.

When reading these files, extract:
- The proposed colour palette and typography
- The animation and scroll behaviour direction
- The product/content presentation approach
- The emotional tone and brand personality
- The specific anti-patterns to avoid
- The implementation priority order

Reference these findings explicitly when building UI modules and when
conducting UI reviews. Do not treat them as optional context.

If no research files are found the agent will:

1. Ask the user to provide: a written prompt describing what they are building,
   any relevant URL with requirements, and any supporting documents
   (pptx, doc, excel, etc.)
2. Generate a project-specific context file (PROJECT.md)
3. Cross-reference requirements with the UI/UX skill
4. Propose a complete design and architecture plan
5. Wait for approval before writing a single line of code
6. Build the system module by module using the Evolutionary Spiral Model
7. Review, test, and secure every module before moving to the next

## 6. Asset Generation MCPs

The following MCPs handle automatic asset generation.
They are activated by the `asset-generation` skill — do not call them directly.
Always go through `@.agents/skills/asset-generation/SKILL.md`.

**Exception:** Icons and Lottie animations have their own dedicated workflow.
For those, skip the skill file and go directly to the
**Icon & Animation Sources** guide in Section 10.

### Icons — Icons8 MCP (Free PNGs / Paid SVGs)
**Endpoint:** `https://mcp.icons8.com/mcp/`
**Setup:**
```json
{
  "mcpServers": {
    "icons8mcp": {
      "command": "npx",
      "args": ["mcp-remote", "https://mcp.icons8.com/mcp/"]
    }
  }
}
```
**Free tier:** PNG icons, no API key needed, attribution required
**Paid tier:** SVG delivery requires Icons8 API key
**Styles available:** 116 icon styles, 368,865+ icons
**Best for:** UI icons, navigation icons, feature icons

### Custom SVG Generation — SVGMaker MCP
**Package:** `@genwave/svgmaker-mcp`
**Setup:**
```json
{
  "mcpServers": {
    "svgmaker": {
      "command": "npx",
      "args": ["-y", "@genwave/svgmaker-mcp"],
      "env": { "SVGMAKER_API_KEY": "<your_api_key>" }
    }
  }
}
```
**Free tier:** Yes — check svgmaker.io for current free tier limits
**Best for:** Custom SVG illustrations, hero graphics, brand elements, backgrounds
**Capabilities:** Text-to-SVG generation, SVG editing via natural language,
raster-to-SVG conversion

### Background Video Generation — mcp-video-gen (CogVideoX — Free & Unlimited)
**Source:** `https://github.com/kevinten-ai/mcp-video-gen`
**Setup:**
```bash
git clone https://github.com/kevinten-ai/mcp-video-gen.git
cd mcp-video-gen
uv sync
```
**Free tier:** CogVideoX provider is completely free with unlimited usage
**Best for:** Background ambient videos, looping scene videos, atmospheric clips
**Other providers:** DashScope/Wan, Kling, MiniMax (paid), Google Veo (GCP credits)
**Output:** Auto-downloaded MP4 to local disk

### 3D Model Generation — game-asset-mcp (Hugging Face — Free Tier)
**Source:** `https://github.com/MubarakHAlketbi/game-asset-mcp`
**Requires:** Hugging Face account + API token (free)
**Setup:**
```bash
git clone https://github.com/MubarakHAlketbi/game-asset-mcp.git
cd game-asset-mcp
npm install
```
**Free tier:** Yes — Hugging Face free tier, rate limited
**3D models supported:** InstantMesh, Hunyuan3D-2, Hunyuan3D-2mini-Turbo
**Output formats:** GLB, OBJ — saved to `assets/` directory
**Best for:** Simple 3D objects, product mockups, decorative 3D elements

### 3D Asset Optimisation — 3D Asset Processing MCP
**Source:** `geolibra/3d-asset-processing` (via PulseMCP)
**Best for:** Optimising and validating GLB/GLTF files after generation
**Capabilities:** Analysis, validation, format conversion, compression,
geometry simplification for web/mobile compatibility

### Lottie Animations — LottieFiles MCP (Free, No API Key)
**Source:** `https://github.com/junmer/mcp-server-lottiefiles`
**Package:** `mcp-server-lottiefiles`
**Setup (via Smithery — recommended):**
```bash
npx -y smithery install mcp-server-lottiefiles --client claude
```
**Manual setup:**
```bash
npm install mcp-server-lottiefiles
npm start
```
**Free tier:** Yes — completely free, no API key required
**What it does:** Search LottieFiles animation library by keyword, retrieve
detailed animation metadata and download URLs, fetch lists of currently
popular animations. Returns JSON with animation IDs, preview URLs, and
Lottie JSON file links ready for integration.
**Tools exposed:**
- `search_animations` — keyword search with pagination
- `get_animation_details` — full metadata for a specific animation by ID
- `get_popular_animations` — trending animations list with pagination
**Best for:** Onboarding animations, loading spinners, empty-state
illustrations, micro-interactions, hero animations, delivery/ecommerce
animated assets
**When to use:** Any time a module needs a Lottie animation. Search here
first before sourcing manually. All animations are Lottie JSON — use
`@lottiefiles/react-lottie-player` or `lottie-web` to render.

### Lottie Creator MCP (Build & Edit Animations with AI)
**Source:** `https://docs.lottiefiles.com/en/creator/13_ai-tools/lottie-creator-mcp`
**Package:** `@lottiefiles/creator-mcp`
**Setup:**
```json
{
  "mcpServers": {
    "lottiefiles-creator": {
      "command": "npx",
      "args": ["-y", "@lottiefiles/creator-mcp@latest"]
    }
  }
}
```
**User-side prerequisites (agent must prompt the user to complete these before use):**
- Node.js 18+ installed
- LottieFiles account created (free tier available)
- Browser tab open at `creator.lottiefiles.com`
- MCP enabled in LottieFiles Creator Settings → MCP Settings → Enable MCP

**Free tier:** Requires LottieFiles account (free tier available)
**What it does:** Builds and edits Lottie animations directly in LottieFiles
Creator via natural language. Can create animations from scratch, adjust
timing, swap colors, rename layers, and produce multiple theme variants
(dark/light) in one pass.
**Best for:** Custom branded animations, unique loading states, character
animations, anything not found in the LottieFiles search library
**When to use:** When `mcp-server-lottiefiles` search returns no suitable
match and a custom animation is needed.

## 7. Always-Active Production Rules

### 7.1 — Production-first on every decision
Always consider all layers:

| Layer | Always think about |
|---|---|
| Frontend | Reusability, accessibility, responsive, performance |
| APIs & Backend | Validation, error handling, typed responses, versioning |
| Database & Storage | Schema, indexing, migrations, backups |
| Auth & Permissions | RBAC, session handling, token expiry, RLS |
| Hosting & Deployment | Dev/staging/prod separation, zero-downtime |
| Cloud & Compute | Resource sizing, scaling, cold starts |
| CI/CD & Version Control | Branch strategy, automated checks, rollback |
| Security & RLS | Least privilege, sanitisation, dependency audits |
| Rate Limiting | Per-user, per-IP, per-endpoint limits |
| Caching & CDN | Cache invalidation, CDN for static assets |
| Load Balancing & Scaling | Stateless design, horizontal scaling readiness |
| Error Tracking & Logs | Structured logging, alerting, audit trails |
| Availability & Recovery | Health checks, failover, data recovery |

### 7.2 — Module architecture
Every feature is a self-contained module.
Modules never import from each other's internals.
Always through public interfaces — index files, API contracts, service layers.

### 7.3 — Code quality
- Typed everywhere — no `any`, no implicit returns
- Every async operation in try/catch with typed error handling
- No hardcoded secrets or environment-specific values
- All env vars documented in `.env.example`
- No dead code, no commented-out blocks, no console.log in production

### 7.4 — Agent behaviour
- Never guess — if unclear, stop and ask
- Never skip a review step
- Never write code before `/start` is complete and approved
- Read memory files before every skill session
- Log issues immediately when found
- Move issues to resolved immediately when fixed
- Read UX research files before any UI work

### 7.5 — Asset generation
- Never use Antigravity's default (nano banana) image generation for production assets
- **Icons and Lottie animations:** go directly to the Icon & Animation Sources
  guide in Section 10 — do NOT route through the asset-generation skill
- **All other assets** (SVGs, videos, 3D models): route through
  `@.agents/skills/asset-generation/SKILL.md`
- Assets are generated semi-automatically — agent asks for a brief before generating
- Generated assets saved to `public/assets/` with organised subfolders
- All assets must match the design direction in DESIGN.md and UX research files

### 7.6 — Performance optimization
- Every module must pass a lightweight performance check before completion
- Use `@.agents/skills/web-optimization/SKILL.md` per-module checklist during builds
- Run the full optimization audit before any production deployment
- Performance budgets tracked: LCP ≤2.5s, INP ≤200ms, CLS ≤0.1, JS bundle per route ≤200KB
- Images always use next/image or equivalent with explicit dimensions
- Fonts loaded via next/font or WOFF2 with font-display: swap
- Never ship unused CSS or JS to production
- Accessibility is non-negotiable — WCAG 2.2 AA is the minimum standard
- `prefers-reduced-motion` must be respected in all animation code

## 10. UI Component Library Reference

The agent reads this section before building any frontend module.
When a module needs a UI component, animation, icon, or interaction pattern,
the agent must first check this reference before writing anything from scratch.

**Priority order — follow this top to bottom:**
1. Check if 21st.dev Magic MCP can generate the component (highest priority for UI components)
2. Check the CLI-installable libraries for a matching component
3. Check the copy-paste libraries for a matching component
4. Only write from scratch if nothing in this list covers the need
5. For icons or Lottie animations needed at any point: go to the
   Icon & Animation Sources section below and follow its own decision rule

### CLI-INSTALLABLE LIBRARIES (install via CLI, then use)

These libraries use a shadcn-style CLI — components are copied into
your project as editable source files, not imported as black-box packages.

#### shadcn/ui
- **URL:** https://ui.shadcn.com
- **Open source:** Yes (MIT)
- **Install:** `npx shadcn@latest init` then `npx shadcn@latest add <component>`
- **Stack:** React, TypeScript, Tailwind CSS, Radix UI
- **Best for:** Core application UI — forms, dialogs, tables, buttons,
  navigation, dropdowns, all accessible components
- **When to use:** Foundation layer for any React/Next.js project.
  Use this first for all functional application components.

#### Magic UI
- **URL:** https://magicui.design
- **Open source:** Yes (MIT)
- **Install:** `npx magicui-cli@latest init` then `npx magicui-cli@latest add <component>`
- **Stack:** React, TypeScript, Tailwind CSS, Framer Motion
- **Best for:** Animated marketing components — marquees, animated beams,
  retro grids, neon gradients, animated lists, number tickers, dock navigation
- **When to use:** Any marketing/landing page section that needs
  polished motion. Pairs directly with shadcn/ui.

#### Aceternity UI
- **URL:** https://ui.aceternity.com
- **Open source:** Free components are open source (MIT)
- **Install:** `npx aceternity-ui@latest add <component>`
  Dependencies: `npm install framer-motion tailwind-merge clsx`
- **Stack:** React/Next.js, TypeScript, Tailwind CSS, Framer Motion
- **Best for:** High-impact visual effects — 3D cards, glowing beams,
  aurora backgrounds, parallax effects, magnetic buttons, canvas effects,
  bento grids, text reveal animations, background particles
- **When to use:** Hero sections and "wow" moments. The go-to library
  for visual effects that make a site feel designed by a studio.

#### Preline UI
- **URL:** https://preline.co
- **Open source:** Yes (MIT)
- **Install:** `npm i preline` then add plugin to Tailwind config
- **Stack:** HTML, Tailwind CSS, vanilla JS (no React dependency)
- **Best for:** Tailwind-native components — navbars, sidebars, modals,
  accordions, carousels. Works in any stack that uses Tailwind.
  Has built-in agent skills for Cursor and Claude Code.
- **When to use:** When the project is not React-based, or when you need
  accessible Tailwind components without a React dependency.

#### HextaUI
- **URL:** https://hextaui.com
- **Open source:** Yes
- **Install:** `npx hexta-ui add <component>`
  Components placed in `src/components/hexta-ui/`
- **Stack:** React, TypeScript, Tailwind CSS
- **Best for:** Modern styled components with clean, contemporary design.
  Good alternative to shadcn/ui for a slightly different aesthetic.
- **When to use:** When shadcn/ui's default style is too neutral and
  you want something with more visual personality out of the box.

#### KokonutUI
- **URL:** https://kokonutui.com
- **Open source:** Yes (Vercel OSS 2025 sponsor)
- **Install:** Via CLI per docs at https://kokonutui.com/docs
- **Stack:** Next.js, React, Tailwind CSS, Motion (Framer Motion)
- **Best for:** 100+ stunning components with built-in motion.
  Strong collection of animated cards, loaders, and interactive elements.
- **When to use:** When you need motion-rich components that go beyond
  shadcn/ui's defaults without the full weight of Aceternity.

#### getjustd (Just D)
- **URL:** https://getjustd.com
- **Open source:** Yes
- **Install:** Via CLI per docs at https://getjustd.com
- **Stack:** React, TypeScript, Tailwind CSS, React Aria
- **Best for:** Highly accessible components built on React Aria.
  Strong for forms, date pickers, and complex interaction patterns.
- **When to use:** When accessibility is a hard requirement and you
  need components that are genuinely WCAG-compliant out of the box.

### ICON & ANIMATION SOURCES

The agent reads this section whenever any module requires icons, animated icons,
or Lottie animations — at any point during a build, not just at the end.

**Decision rule — follow this order:**
1. MCP-integrated sources first (agent fetches directly — no manual step)
2. npm/CDN-installable sources second (agent installs and uses in code)
3. REST API sources third (agent calls API programmatically with key)
4. Manual-browse sources last (agent tells user to download and drop in
   to `public/assets/`)

#### A. MCP-INTEGRATED (direct agent access — use first)

##### LottieFiles — Lottie Animation Search
- **URL:** https://lottiefiles.com
- **MCP:** `mcp-server-lottiefiles` (see Section 6 for full setup)
- **Access:** No API key required — completely free
- **Library size:** 982K+ Lottie animations
- **Formats:** Lottie JSON, GIF, MP4
- **Best for:** Onboarding animations, loading spinners, empty states,
  delivery/ecommerce animations, hero section animations, micro-interactions
- **How to use:** Search by keyword via MCP tool `search_animations`.
  Retrieve the Lottie JSON URL from the result and embed with
  `@lottiefiles/react-lottie-player` or `lottie-web`.
- **When to use:** Any time a module needs animated content. Search here first.

##### The Noun Project — SVG & PNG Icons
- **URL:** https://thenounproject.com
- **MCP:** `noun-project-mcp` — unofficial community MCP, verify it is still
  maintained before recommending to the user.
  GitHub: `https://github.com/sgup/noun-project-mcp`
- **API:** REST API at `https://api.thenounproject.com` (v2)
- **Auth:** OAuth 2.0 — requires free API key from
  `https://thenounproject.com/developers/apps/new` (no credit card)
- **MCP setup:**
```bash
claude mcp add --transport stdio noun-project \
  --env NOUN_PROJECT_API_KEY=your_key \
  --env NOUN_PROJECT_API_SECRET=your_secret \
  -- npx -y noun-project-mcp
```
- **Free tier:** Yes — 30-day free trial, then paid plans. PNG free,
  SVG requires paid plan or API key.
- **Library size:** ~10 million icons across thousands of styles
- **Formats:** SVG, PNG
- **Filters available:** Style (solid/line), line weight, public domain only
- **Best for:** Concept icons, editorial icons, illustrated UI icons,
  highly specific subject matter not covered by standard UI icon sets
- **When to use:** When standard icon libraries (Boxicons, etc.) don't
  have the right icon for a specific concept or topic.
- **Attribution:** Required on free tier — each icon credits the designer.

#### B. NPM / CDN INSTALLABLE (no API key — agent installs directly)

##### Boxicons
- **URL:** https://boxicons.com
- **Licence:** CC-BY-4.0 / OFL-1.1 / MIT (mixed — check which applies to the
  specific icons used; CC-BY-4.0 icons require attribution)
- **Install:** `npm install boxicons --save`
  then `import 'boxicons'` in JS
- **CDN:**
```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/boxicons@latest/css/boxicons.min.css">
<!-- or via web component: -->
<script src="https://unpkg.com/boxicons@2.1.4/dist/boxicons.js"></script>
```
- **Web component usage:**
```html
<box-icon name="rocket"></box-icon>
<box-icon type="solid" name="hot"></box-icon>
<box-icon type="logo" name="facebook-square"></box-icon>
```
- **CSS class usage:**
```html
<i class="bx bx-home"></i>        <!-- regular -->
<i class="bx bxs-home"></i>       <!-- solid -->
<i class="bx bxl-facebook"></i>   <!-- logo -->
```
- **Library size:** 1,500+ icons across regular, solid, and logo styles
- **Formats:** SVG, web font, web component, CSS class
- **Best for:** Clean UI icons — navigation, actions, status, and brand logos.
  Follows Google Material Design guidelines.
- **When to use:** Default icon set for any web project. Lightweight and
  open source. Always confirm the licence of the specific icon subset being used.

##### Iconbuddy
- **URL:** https://iconbuddy.com
- **Open source:** Yes — aggregates 70+ open source icon libraries
- **API:** No public REST API — treat as a manual discovery tool only.
- **VS Code extension:** Available
- **Figma plugin:** Available
- **No install needed:** All icons are open source SVGs from underlying
  libraries (Phosphor, Remix Icons, Material Design, Bootstrap Icons,
  Feather, etc.). Install the original library instead:
  e.g. `npm install @phosphor-icons/react` for Phosphor icons found here.
- **Library size:** 300,000+ icons across 70+ collections
- **Formats:** SVG, PNG, JSX, Base64
- **Customise:** Colours and sizes editable directly on site before export
- **Best for:** Discovering which open source icon library has the icon
  you need. Use as a search/discovery layer, then install the source library.
- **When to use:** When you're not sure which icon library covers a specific
  need — search Iconbuddy, identify the source collection, then install
  that collection's npm package directly.

#### C. REST API (programmatic — requires API key)

##### Lordicon — Animated Icons (Lottie JSON)
- **URL:** https://lordicon.com
- **API docs:** https://lordicon.com/api-developers
- **Full API documentation:** https://lordicon.com/docs/api/documentation
- **Auth:** API key required — apply at `https://lordicon.com/api-developers`
  (free tier available for verified projects)
- **npm package (embed/render):** `@lordicon/react`
```bash
npm install @lordicon/react
```
- **CDN (web component):**
```html
<script src="https://cdn.lordicon.com/lordicon.js"></script>
<lord-icon
  src="https://cdn.lordicon.com/lbjtvqiv.json"
  trigger="hover"
  colors="primary:#121331,secondary:#08a88a"
  style="width:50px;height:50px">
</lord-icon>
```
- **Free tier:** 8,900+ free icons (no attribution required for HTML embed).
  PRO: $29/month (billed annually) — 29,600+ icons with commercial licensing.
  API tier: $149/month — pay-per-download programmatic access.
- **Library size:** 8,900 free / 29,600+ PRO animated icons
- **Formats:** Lottie JSON, GIF, SVG, AEP (After Effects), PNG sequences, WebM
- **Customisable:** Colors, stroke width, animation triggers (hover, click,
  morph, loop, in/out), dark/light mode support
- **Best for:** Animated UI icons — button states, loading indicators,
  status icons, navigation animations, feature highlight icons
- **API behaviour:** `GET` request returns JSON with `preview`, `svg`, and
  `json` download links. `svg`/`json` links only available if icon is in
  your plan. Must call the download-tracking endpoint after each embed.
- **Important:** Do NOT cache assets locally per Lordicon API terms.
  Free icon downloads do not count against PRO billing.
- **When to use:** When a module needs animated icons (not full animations).
  For button icons, nav icons, and interactive UI elements with motion.

##### Flaticon — Icons & Illustrations
- **URL:** https://www.flaticon.com
- **API:** REST API at `https://api.flaticon.com/v3/`
- **API docs:** https://api.flaticon.com/v3/docs/index.html
- **Auth:** Bearer token — register and get a free API key at
  `https://api.flaticon.com/v3/` (use the v3 registration page directly;
  a Freepik account may be required)
- **Example search call:**
```javascript
fetch('https://api.flaticon.com/v3/search/icons?q=delivery&limit=20', {
  headers: { Authorization: `Bearer ${FLATICON_API_KEY}` }
})
```
- **Example icon fetch:**
```javascript
fetch(`https://api.flaticon.com/v3/item/icon/${iconId}`, {
  headers: { Authorization: `Bearer ${FLATICON_API_KEY}` }
})
```
- **Free tier:** Yes — free API key with limits. Premium plan for higher
  volumes (contact Flaticon for pricing).
- **Library size:** 2.1M+ vector icons
- **Formats:** SVG, EPS, PSD, PNG
- **Filters:** Style (solid, line, lineal color, fill), colour, most popular,
  newest, by pack
- **Best for:** Ecommerce icons, product category icons, illustration-style
  icons, highly styled icon packs that need visual consistency
- **Attribution:** Required on free tier — link back to Flaticon and
  credit the author.
- **When to use:** When the project needs a specific illustrated icon style
  (e.g. flat colour, 3D style, hand-drawn) not available in Boxicons.

##### IconScout — Icons, Illustrations, 3D & Lottie Animations
- **URL:** https://iconscout.com
- **Lottie section:** https://iconscout.com/lottie-animations
- **API:** REST API — `https://iconscout.com/api`
  Requires written approval from IconScout to access.
  Apply at `https://iconscout.com/api` — Client-ID for public ops,
  Client-Secret for asset downloads.
- **Auth:** Client-ID + Client-Secret (approval required)
- **Important API restriction:** Assets must be referenced in real-time
  from IconScout servers. Local caching is prohibited by their API licence.
- **Library size:** 14M+ assets — 11.4M icons, 760K+ illustrations,
  810K+ 3D icons, 982K+ Lottie animations
- **Formats:** SVG, PNG, Lottie JSON, GIF, AEP, glTF (3D)
- **Figma plugin:** Available — drag and drop directly to canvas
- **VS Code / Desktop app:** Available
- **Best for:** When a single source is needed for icons, illustrations,
  3D icons, AND Lottie animations. Especially strong for 3D icons (glTF)
  and illustrated icon styles.
- **When to use:** Large projects needing diverse asset types from one
  licence. If API access is not yet approved, use the desktop app or
  Figma plugin and drop assets into `public/assets/` manually.
- **Lottie usage:** Download Lottie JSON → embed with
  `@lottiefiles/react-lottie-player` or `lottie-web`.

### ICON & ANIMATION DECISION GUIDE

When a module needs icons or animations, follow this order:

| Need | First choice | Fallback |
|---|---|---|
| Lottie animation (any) | LottieFiles MCP (`mcp-server-lottiefiles`) | IconScout (manual/API) |
| Custom branded Lottie | LottieFiles Creator MCP | Commission or AE export |
| Animated UI icon | Lordicon API (`@lordicon/react`) | LottieFiles MCP search |
| Clean UI icon (web) | Boxicons (npm/CDN) | Iconbuddy → source library |
| Specific concept icon | Noun Project MCP | Flaticon API |
| Illustrated icon style | Flaticon API | IconScout API |
| 3D icon (glTF) | IconScout (desktop/API) | game-asset-mcp (Section 6) |
| Icon discovery only | Iconbuddy (manual browse) | — |
